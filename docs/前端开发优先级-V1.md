# InkMuse 前端开发优先级（V1）

## 1. 背景与目标

- 背景：后端 V1 能力已闭环，前端 V1 主流程已落地，当前进入收尾与联调验收阶段。
- 目标：在 `Vite + React + TypeScript` 技术栈下，交付“项目创建 -> 资产管理/生成 -> 对话微调 -> 章节生成与当前稿确认”的可用闭环。
- 范围：仅 V1，不包含一致性守卫、导出、成本看板。
- 状态口径：`已完成` = 已落地代码并通过 `lint/build`；`未完成` = 尚未实现或尚未完成联调验收。

## 3. API 对接约束

- 统一错误响应：`{ "error": "..." }`。
- 错误码策略：
  - `400`：输入校验失败。
  - `404`：对象不存在。
  - `409`：并发冲突（前端需提示重试）。
- 当前稿确认：请求头必须包含合法 UUID 的 `X-User-ID`。

## 4. 前端实现约束

- 设计语言：严格遵循 `UI.txt` 的扁平化规范（无阴影、强色块、几何布局、Outfit 字体）。
- 状态管理：React Query 管理服务端状态，表单使用 `react-hook-form + zod`。
- 技术约束：
  - `VITE_API_BASE_URL` 作为唯一 API 入口配置。
  - 默认 `http://127.0.0.1:8080/api/v1`。
- 用户标识策略：
  - 首次访问使用 `crypto.randomUUID()` 生成 `user_id`，保存在 `localStorage`。
  - 所有“确认当前稿”请求透传该值。
