# InkMuse Frontend

InkMuse 前端基于 `Vite + React + TypeScript + Tailwind CSS`。

## 当前状态

- P0 主流程已落地：灵感对话创建项目、资产 CRUD 与 AI 生成、项目/资产对话微调（含确认写回）、章节生成/续写/局部改写/当前稿确认。
- 布局已重构为 **侧边栏 + 工作区** 模式（类似 Notion/Obsidian），侧边栏集成项目列表与快捷导航。
- 首页提供「灵感对话」入口：输入一句话灵感，AI 自动构思故事框架并创建项目。
- AI 资产生成已提供 `generation_record.status` 的显式状态展示（运行中/成功/失败）。
- **Tiptap 富文本编辑器** 已集成：章节内容支持富文本编辑（标题、加粗、斜体、列表、引用），选中文本可直接触发 AI 改写，草稿章节支持手动保存，已确认章节为只读模式。
- **结构化设定卡片** 已实现：角色资产提供 DNA 表单（姓名/年龄/性格标签/动机/外貌/口头禅等），世界观资产提供分区表单（地理/政治/魔法体系/科技水平等），支持结构化/原始文本双模式切换，向后兼容旧版纯文本资产。
- **导出中心** 已实现：工作台标题栏提供导出按钮，支持 Markdown 和纯文本两种格式下载。

## 快速启动

1. 安装依赖

```bash
npm install
```

2. 配置环境变量

```bash
cp .env.example .env
```

默认后端地址：`http://127.0.0.1:8080/api/v1`

3. 启动开发服务

```bash
npm run dev
```

## 可用脚本

- `npm run dev`：本地开发
- `npm run build`：生产构建
- `npm run preview`：预览构建产物
- `npm run lint`：ESLint 检查
- `npm run test`：Vitest 单测
- `npm run test:e2e`：Playwright E2E

## 目录结构

```text
src/
├── app/                    # 路由与全局 Provider
├── pages/                  # 页面层（工作台、设置、404）
├── features/               # 业务模块
│   ├── inspiration/        #   灵感对话创建项目
│   ├── assets/             #   资产管理与 AI 生成
│   │   ├── components/     #     结构化表单（角色/世界观）与展示组件
│   │   └── schemas/        #     Zod 校验 schema 与内容格式工具
│   ├── conversations/      #   对话微调
│   ├── chapters/           #   章节生成
│   │   ├── components/     #     Tiptap 编辑器、工具栏、改写弹窗
│   │   └── lib/            #     纯文本 ↔ Tiptap JSON 转换器
│   ├── projects/           #   项目编辑面板
│   ├── prompts/            #   Prompt 模板
│   └── llm-providers/      #   LLM 配置
└── shared/                 # API、配置、工具、UI 基础组件
    └── ui/                 #   Sidebar、AppShell、Button、Card 等
```

## 布局架构

采用侧边栏 + 工作区布局，AppShell 在路由层统一包裹：

- **侧边栏**：Logo、新灵感按钮、项目列表（可滚动）、设置入口、折叠切换
- **工作区**：根据路由渲染灵感对话页 / 项目工作台 / 设置页

## 路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | InspirationPage | 灵感对话创建项目 |
| `/projects/:id` | ProjectWorkbenchPage | 项目工作台（设定/对话/章节/Prompt Tab 切换） |
| `/settings` | SettingsPage | LLM Provider 全局设置 |

## 拓展方向

> 基于 `AI小说项目最佳实践方案.md` 中前端交互层的最佳实践，对照 InkMuse V1 前端当前覆盖情况，规划分阶段演进路线。

### 对齐评估（V1 vs 最佳实践）

| 最佳实践模块 | V1 覆盖度 | 现状说明 |
|---|---|---|
| 沉浸式编辑器 | **已覆盖** | Tiptap 富文本编辑器已集成，支持基础排版（标题/加粗/斜体/列表/引用）与选中文本 AI 改写；Ghost Text 续写建议待后续迭代 |
| 设定管理面板 | **已覆盖** | 角色 DNA 结构化表单、世界观分区表单已落地；支持结构化↔原始文本双模式切换，向后兼容旧版纯文本 |
| 知识图谱可视化 | 未覆盖 | 无角色关系 / 实体关系可视化 |
| 伏笔 / 时间线可视化 | 未覆盖 | 无伏笔追踪与时间线视图 |
| 灵感工作台 | 基础覆盖 | 单轮灵感对话创建项目已完成；无多轮 ReAct Agent、无跨项目引用 |

### 第一阶段（V1.1）：富文本编辑器 + 设定结构化 ✅ 已完成

1. **Tiptap 富文本编辑器** ✅ — 已替代纯文本展示，提供基础排版能力（标题、段落、加粗、斜体、列表、引用、撤销/重做）；选中文本后可直接触发后端局部重写接口进行 AI 改写；草稿章节支持手动保存，已确认章节自动切换为只读模式。
2. **Ghost Text 续写建议** — 待后续迭代：基于 Tiptap Extension 实现光标位置续写预览（灰色虚文本），Tab 键接受。
3. **结构化设定卡片** ✅ — 角色资产提供 DNA 表单（姓名 / 年龄 / 性别 / 性格标签 / 动机 / 外貌 / 口头禅 / 背景故事 / 人物关系 / 备注）；世界观资产提供分区表单（地理 / 政治 / 魔法体系 / 科技水平 / 文化 / 历史 / 经济 / 宗教 / 备注）；支持结构化↔原始文本双模式切换，向后兼容旧版纯文本资产（可一键迁移，原文移入备注字段）；大纲资产保持纯文本编辑。

### 第二阶段（V1.2）：可视化 + 质量反馈

1. **人物关系图谱** — 基于 React Flow 实现知识图谱可视化，展示角色之间的关系（盟友 / 敌对 / 亲属 / 师徒等）；支持拖拽编辑节点与关系，数据同步至后端知识图谱表。
2. **伏笔时间线** — 横轴为章节序列，纵轴为伏笔条目；用颜色标记伏笔状态（已埋设 / 已回收 / 逾期未回收）；点击伏笔可跳转到相关章节。
3. **章节质量评审面板** — 展示后端质量评审结果：多维度评分（逻辑一致性 / 角色还原度 / 节奏感 / 文笔质量）+ 具体问题列表与定位。
4. **AI 修改对比视图** — 基于 Tiptap diff 扩展实现修改高亮（新增绿色 / 删除红色），支持逐段接受或拒绝 AI 建议的修改。

### 第三阶段（V2）：灵感工作台进化 + 高级交互

1. **灵感工作台增强** — 支持多轮对话式灵感探索（ReAct Agent 模式，AI 主动提问引导创作方向）；支持跨项目引用已有设定与角色。
2. **等级体系编辑器** — 树形结构编辑器，用于管理修仙 / 武侠 / 奇幻等级体系（境界名称 / 能力描述 / 晋级条件），数据作为世界观子资产存储。
3. **消痕结果对比视图** — 并排展示原始 AI 生成文本与消痕后文本，高亮标注修改位置，支持选择性采纳。
4. **角色状态仪表盘** — 以时间轴形式展示角色的位置变化、情绪曲线、关系变化轨迹，数据来源于后端角色状态表。
